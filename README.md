# Politique de confidentialité de UserFinder
Dernière mise à jour : 19 juillet 2026
1. Présentation
UserFinder est un bot Discord conçu pour afficher des informations publiques sur les utilisateurs et les serveurs via l'API officielle de Discord.
2. Données collectées
Selon les commandes utilisées, UserFinder peut accéder aux données suivantes :
ID utilisateur
Nom d'utilisateur et pseudo
Avatar
ID du serveur
ID des salons
ID des rôles
Date de création du compte
Date d'arrivée sur le serveur
Rôles visibles sur le serveur
Informations publiques fournies par l'API Discord
Le bot ne collecte que les données nécessaires à son fonctionnement. Les développeurs de bots Discord sont tenus d'expliquer clairement quelles données sont utilisées et dans quel but. �
Discord +1
3. Données non collectées
UserFinder ne collecte pas et ne peut pas accéder à :
Mots de passe
Adresses e-mail
Adresses IP
Messages privés (DM)
Contenu privé non accessible via l'API Discord
Moyens de paiement
4. Utilisation des données
Les informations récupérées servent uniquement à :
Exécuter les commandes demandées.
Afficher des informations publiques sur Discord.
Assurer le bon fonctionnement du bot.
5. Partage des données
UserFinder ne vend, ne loue et ne partage aucune donnée personnelle avec des tiers.
6. Conservation des données
Par défaut, UserFinder ne stocke aucune donnée personnelle de manière permanente, sauf si une fonctionnalité spécifique du bot l'exige (par exemple un système de modération ou de configuration).
7. Sécurité
Nous mettons en œuvre des mesures raisonnables afin de protéger les données traitées contre tout accès non autorisé.
8. Vos droits
Vous pouvez demander la suppression des données éventuellement stockées par UserFinder en contactant le développeur.
9. Services tiers
UserFinder fonctionne grâce à l'API de Discord. L'utilisation du bot implique également l'application de la politique de confidentialité de Discord. �

Cette politique peut être mise à jour à tout moment. Les modifications prennent effet dès leur publication.
